# 正式公開前チェックリスト

最終更新: 2026-06-12

このアプリは Android（Chrome）・iPhone（Safari）・PC（Chrome/Edge）の3環境で
動作することを想定している。公開前に各環境で以下を確認する。

---

## 0. 前提・先に確認しておくこと

- [ ] 公開URLが **HTTPS** であること（カメラ起動・PWAインストールはHTTPS必須）
- [ ] テスト用ログインアカウントを用意し、各端末で同じ手順を再現できるようにする
- [ ] **データはデバイスごとに独立している点を理解する**
  - プロジェクト・写真メタ情報・コメントは `localStorage`（zustand persist）
  - 写真本体（元画像・600px圧縮版）は `IndexedDB`（idb-keyval）
  - ログインのみ Supabase（クラウド）。写真・プロジェクトデータは**端末間で同期されない**
  - 要件定義 8.1 の「同一アカウントでスマホ・PC両方から使える」という想定とギャップがあるため、
    公開時にユーザーへどう案内するか方針を決めておく（[[project_roadmap]] 参照）

---

## 1. 共通機能チェック（3環境すべてで実施）

### 1.1 認証
- [ ] 新規登録ができる
- [ ] ログイン・ログアウトができる
- [ ] リロード後もログイン状態が保持される
- [ ] 未ログイン状態で保護ページにアクセス→ログイン画面にリダイレクトされる

### 1.2 プロジェクト管理
- [ ] 新規プロジェクト作成・編集・削除
- [ ] 一覧の枚数・更新日時表示が正しい

### 1.3 写真アップロード
- [ ] 複数枚同時アップロードができ、進捗表示が正しい
- [ ] アップロード時の一括フェーズ設定（施工前/施工中/施工後）が反映される
- [ ] **HEIC形式の写真**をアップロードした場合の表示を確認（環境により下記の挙動差あり）
  - サムネイル/ライトボックスで正しく表示されるか、プレースホルダー（頭文字アイコン）になっていないか
  - Excel出力時に画像が埋め込まれるか（HEICはCanvasで読み込めない環境だと画像が欠落する）

### 1.4 写真表示・操作
- [ ] グリッド表示・ライトボックス表示
- [ ] フェーズタブ切り替え（全て/施工前/施工中/施工後/未分類）
- [ ] 写真の並べ替え（ドラッグ）
- [ ] 個別フェーズ変更
- [ ] 複数選択 → 一括フェーズ変更・一括削除・「未分類」へのクリア（BatchActionBar）

### 1.5 台帳ビュー
- [ ] 台帳ビューへの切り替え
- [ ] コメント入力 → 自動保存 → リロードで保持されている
- [ ] 3段階コメントテンプレートの挿入

### 1.6 Excel出力
- [ ] 通常出力（表紙・写真台帳・A4印刷レイアウト・画像埋め込み）
- [ ] 施工前後出力（BeforeAfterExportButton）
- [ ] フェーズ別シート分割・連番が正しい

### 1.7 施工前後オーバーレイ撮影
- [ ] 施工前写真を選んでオーバーレイ表示できる
- [ ] 透明度スライダー、表示/非表示トグルが動作する
- [ ] ズーム＋/−・リセットボタンが動作する
- [ ] 撮影 → 「施工後」フェーズで保存される、連続撮影枚数が表示される
- [ ] 「施工前写真を変更」「完了」ボタンの動作

### 1.8 PWA共通
- [ ] manifest.webmanifest が読み込まれ、アイコン・アプリ名「現場フォト」が正しい
- [ ] 本番ビルド（`vite build` + `preview`等）で Service Worker が登録される
  - `import.meta.env.PROD` 判定のため、`vite dev` では登録されない点に注意

## 2. Android（Chrome）固有チェック

- [ ] Chromeメニュー「アプリをインストール」/インストールバナーから追加できる
- [ ] ホーム画面アイコン・アプリ名が正しい
- [ ] インストール後、スタンドアロン表示（URLバーなし、theme-color #18181b が反映）で起動する
- [ ] オーバーレイ撮影で背面カメラ（`facingMode: environment`）が起動する
- [ ] オーバーレイ操作：1本指ドラッグで移動、2本指ピンチで拡大縮小ができる
- [ ] 通常の写真アップロードで「カメラで撮影」「ギャラリーから選択」両方を選べる
- [ ] 画面回転（縦⇔横）でレイアウトが崩れない
- [ ] 端末の戻る操作でオーバーレイ撮影・各モーダルが意図通りに閉じる
- [ ] Excel出力後、ダウンロードフォルダに保存され、Sheet等で開ける
- [ ] ブラウザ再起動後もプロジェクト・写真・コメントが保持されている
- [ ] 写真を多数（元画像＋圧縮版）保存してもIndexedDBの容量エラーが出ない

## 3. iPhone（Safari）固有チェック

- [ ] 共有メニュー →「ホーム画面に追加」ができる
- [ ] ホーム画面アイコン（apple-touch-icon）・名称（現場フォト）が正しい
- [ ] ホーム画面起動でスタンドアロン表示（Safari UIなし）になる
- [ ] ノッチ/Dynamic Island機種でもオーバーレイ撮影の上部バー・下部操作バーが
      `env(safe-area-inset-*)` で隠れずに表示される
- [ ] カメラアクセスの許可ダイアログが表示され、許可後にプレビューが映る
- [ ] `<video playsInline muted autoPlay>` が自動再生される（手動再生待ちにならない）
- [ ] iPhone標準カメラで撮影した**HEIC写真**をそのままアップロードできる
  - サムネイル・ライトボックス・台帳・Excel出力すべてで正しく表示されるか
- [ ] オーバーレイ撮影でピンチズーム・1本指ドラッグが正しく動作する
- [ ] Excel出力後、「ファイル」アプリまたは共有シートから保存・開ける
- [ ] ホーム画面アプリを数日空けて再起動しても、プロジェクト・写真データが消えていない
      （iOSのストレージ自動削除リスクの確認）

## 4. PC（Chrome/Edge）固有チェック

- [ ] アドレスバーのインストールアイコンからアプリとしてインストールできる
- [ ] ウィンドウ幅 1024px（lg breakpoint）でサイドバー表示⇔ボトムナビ表示が切り替わる
- [ ] 写真アップロードでドラッグ＆ドロップが機能する
- [ ] オーバーレイ撮影でWebカメラが起動する
  - 背面カメラがない場合に `video: true` のフォールバックで内蔵カメラが起動するか
- [ ] マウス操作でオーバーレイのドラッグ移動、＋/−ボタンでの拡大縮小ができる
  （PCではピンチ操作不可のためボタン操作で代替できることを確認）
- [ ] Excel出力ファイルをExcel/Googleスプレッドシート等で開き、画像・レイアウト崩れがない
- [ ] ブラウザ印刷プレビューでA4写真台帳レイアウトが正しく出力される
- [ ] キーボード操作（Tab移動・Enter確定）でフォーム入力ができる

## 5. 公開前に方針決定が必要な項目

- [ ] **デバイス間データ非同期**について、ユーザー向けにどう案内するか
      （現状は1台＝1データ。複数端末で使う運用ならその旨を明記する）
- [ ] HEIC画像がAndroid/PCの一部ブラウザで表示・Excel埋め込みできない場合の案内文言
- [ ] ブラウザの「閲覧データ削除」でプロジェクト・写真が消えるリスクの案内
- [ ] 公開URLがHTTPSであることの最終確認（カメラ・PWAインストールの前提条件）
