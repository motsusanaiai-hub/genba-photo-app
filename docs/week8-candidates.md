## 1. Excel 出力テンプレート複数化

### テンプレート案

#### ① 標準台帳（現行）
- 2列 × 3段、1ページ 6 枚
- 写真 + コメント
- 実装済み

#### ② 報告書型
- 左：写真大（列幅広め）、右：情報欄
- 縦 3 段、1ページ 3 枚
- 記載項目：区分、部位、状況、対応内容

#### ③ 施工前後型
- 左列：施工前、右列：施工後 の対比固定レイアウト
- 縦 3 段、1ページ 6 枚（Before/After ペア × 3）
- フェーズ `before` / `after` を自動マッピング

---

### 実装難易度

| テンプレート | 難易度 | 理由 |
|------------|--------|------|
| ① 標準台帳 | — | 実装済み |
| ② 報告書型 | 中 | 列幅・セル結合の変更。情報欄の入力項目をどこに持つかが設計の要 |
| ③ 施工前後型 | 低〜中 | ソート済み写真を before/after でペアリングするロジックが必要。レイアウト自体は標準台帳の派生 |

### 優先順位

`③ 施工前後型` > `② 報告書型`

- 施工前後型は補修工事の提出書類として需要が高く、現行フェーズタグと相性が良い
- 報告書型は「部位」「状況」などの追加入力項目が必要になるため、データモデルの変更を伴う

### 設計方針

```
generateExcel(project, photos, template: 'standard' | 'report' | 'before-after')
```

- `template` パラメータでシート構築関数を切り替える
- `buildPhotoSheet` を template ごとに分離（`buildStandardSheet` / `buildReportSheet` / `buildBeforeAfterSheet`）
- レイアウト定数（`COL_W`, `H_IMAGE` 等）は template ごとに独立したオブジェクトで持つ
- ExportButton にテンプレート選択 UI（ドロップダウン or ラジオ）を追加

### 将来的な拡張性

- テンプレート定義をオブジェクトとして外出しすることで、追加が 1 ファイル追記で完結する構成にする
- 将来的にユーザーがカスタムテンプレートを保存できる拡張も視野に入れる

## 2. 写真一覧の一括操作

### 機能案

#### ① 複数選択削除
- 写真を複数選択してまとめて削除
- 削除対象：原本（IndexedDB）+ 圧縮版（IndexedDB）+ メタデータ（Zustand/localStorage）
- 削除前に確認ダイアログ（「X 枚の写真を削除しますか？」）

#### ② 複数選択タグ（フェーズ）一括変更
- 複数選択して施工前 / 施工中 / 施工後 を一括設定

### 想定 UI

```
[ □ 写真1 ]  [ □ 写真2 ]  [ □ 写真3 ]  ...

選択中 3 枚  ─────────────────────────────────────
[ 削除 ]  [ 施工前 ]  [ 施工中 ]  [ 施工後 ]  [ 選択解除 ]
```

- 各 PhotoCard にチェックボックスを追加（通常時は薄表示、ホバーで出現）
- 1 枚以上選択中は画面下部に一括操作バーを表示（固定フッター形式）
- 選択状態は `usePhotoSelection` カスタムフックで管理（Zustand には持たない）

### 実装難易度

| 機能 | 難易度 | 理由 |
|------|--------|------|
| 複数選択UI（チェックボックス） | 低 | ローカル state で完結。PhotoCard の変更のみ |
| 一括削除 | 低 | `usePhotos.removePhoto` の並列呼び出し。確認ダイアログは既存 UI で実装可能 |
| 一括タグ変更 | 低 | `usePhotos.setPhase` の並列呼び出しのみ |
| 一括操作バー UI | 低〜中 | 固定フッターの z-index 管理、モバイル対応が必要 |

### 優先順位

`一括操作（削除＋タグ変更）` を Week 8 の最優先候補とする。

- 難易度が低く、写真枚数が多い現場で即効性が高い
- データモデルの変更不要
- テンプレート追加より先に実装できる

### 設計方針

```typescript
// usePhotoSelection.ts（新規フック）
const { selected, toggle, selectAll, clear } = usePhotoSelection()

// 一括削除
await Promise.all(selected.map(id => removePhoto(id)))

// 一括タグ変更
selected.forEach(id => setPhase(id, phase))
```

- 選択状態はコンポーネントローカルに持ち、ストアに混入させない
- `PhotoGrid` が選択状態を管理し、`PhotoCard` に `isSelected` / `onToggle` を props で渡す
- 一括操作バーは `PhotoGrid` の外側（`ProjectPage` レベル）に配置してレイアウト干渉を避ける

### 将来的な拡張性

- 選択機能が整備されれば「選択した写真だけ Excel 出力」「選択した写真を別プロジェクトへ移動」などに応用できる
- ドラッグ選択（範囲選択）は実装コストが高いため後回し
