# 収益直結機能の優先順位

作成日: 2026-05-31

---

## 考え方：機能は3つの役割に分類できる

```
集客機能     → 無料ユーザーを増やす（間接的に収益へ）
転換機能     → 無料→Proへ押し上げる（直接収益に直結）
法人化機能   → Pro→法人へ押し上げる（最大単価）
```

---

## P0：転換の核（これがないと誰も払わない）

**Excel出力 ＋ 削除時自動行詰め・コメント連動削除**

この2つは事実上セット。Excel出力単体では既存ソフトと差がない。
「削除したら自動で整形される」という体験が初めて「これは違う」と感じさせる瞬間になる。

収益への影響：**Proプランへの転換率を決定づける唯一の機能。**
ここを後回しにすると、他の有料機能がどれだけ揃っても転換が起きない。

---

## P1：転換の後押し（あると転換速度が上がる）

| 機能 | 役割 | 理由 |
|---|---|---|
| 写真自動圧縮 | 即効性のある体験価値 | アップロードするたびに恩恵を感じる。毎日使う機能 |
| PDF出力 | 提出書類の完結 | Excel不要な現場でもPDFは使う。転換のもうひとつの動機になる |

この2つは「Proにしたら台帳作成が全部終わる」という感覚を完成させる。

---

## P2：法人化のトリガー（Proから法人へ引き上げる）

| 機能 | 役割 | 理由 |
|---|---|---|
| Google Drive連携 | チーム共有の入口 | 個人がDropbox代わりに使い始め、「会社のDriveにも保存したい」という需要が法人契約の前兆になる |
| 複数ユーザー共有 | 法人プランの本体 | これを必要とした瞬間が法人プランへの転換点 |
| 権限管理 | 法人契約の必要条件 | 会社として契約する際に管理者が求める機能 |

---

## P3：集客の核（無料ユーザーを獲得・維持する）

| 機能 | 役割 | 理由 |
|---|---|---|
| ガイドオーバーレイ撮影 | 口コミの発生源 | 「同じ画角で撮れる」体験は現場で見せると驚かれる。現場の口コミは最強の集客 |
| 施工前後比較 | 継続使用の動機 | 写真が蓄積されるほど価値が増す。データが溜まるとやめにくくなる |

これらはすでにMVPに含まれており、**集客コストゼロで口コミを生む設計**になっている。

---

## P4：あると良いが後回しでよい

| 機能 | 理由 |
|---|---|
| 工事看板挿入 | 差別化にはなるが、台帳作成の本質ではない |
| 管理者ダッシュボード | 法人契約後の満足度向上。契約獲得には直接不要 |
| 独自テンプレート共有 | 法人の継続率を上げる機能。獲得フェーズでは不要 |

---

## 開発順序の推奨

```
Phase 1（MVP）
  ガイドオーバーレイ ＋ 施工前後比較 ＋ タグ管理
  → 無料ユーザーを獲得し、口コミを発生させる

Phase 2（最優先）
  Excel出力 ＋ 削除時自動行詰め・コメント連動削除
  → 初めてお金が入る。ここが事業の起点

Phase 3
  写真自動圧縮 ＋ PDF出力
  → 転換速度を上げる。Proの「全部入り」感を完成させる

Phase 4
  Google Drive連携 ＋ 複数ユーザー共有 ＋ 権限管理
  → 法人収益の柱を作る

Phase 5
  工事看板挿入 ＋ 管理者ダッシュボード ＋ テンプレート共有
  → 満足度・継続率の向上
```

**Phase 2のExcel出力が最速で収益化できるか否かを決める。**
ここの完成度（特に削除時の自動整形）が競合との差になるため、工数を惜しまない判断が必要。
