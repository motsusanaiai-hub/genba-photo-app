# 工事写真管理アプリ 要件定義書

最終更新: 2026-05-31（MVP再設計に伴い全面改訂）

---

## 1. プロダクト概要

### 1.1 プロダクトの本質

このアプリは「工事写真アプリ」ではない。

**「工事写真台帳を楽に作るアプリ」** である。

現場の職人・監督が抱える本当の課題は「写真を撮ること」ではなく、
**「撮った写真に説明を付けて台帳にまとめて提出すること」** にある。

コメント入力の時間削減が最大の価値提供。

### 1.2 ターゲットユーザー

| ユーザー | 主な課題 |
|---|---|
| 職人・一人親方 | スマホで撮った写真を整理して提出する手間 |
| 現場監督 | 提出書類の作成に取られる時間（コメント入力・Excel整形） |
| 中小建設会社 | 専用ソフト導入コスト・習熟コストの高さ |

### 1.3 差別化ポイント

- **コメント入力の最速化**：台帳ビュー + テンプレートにより、30枚のコメント入力を40分→10分に短縮
- **Excel出力の自然さ**：専用ソフト形式ではなく誰でも開けるExcelファイルとして出力。写真削除時の自動行詰めが他社にない強み
- **無料で使い始められる**：職人・一人親方が試しやすい料金設計

## 2. 機能要件

### 2.1 無料プラン機能

| 機能 | 説明 |
|---|---|
| プロジェクト管理 | 工事案件単位での写真管理（5件まで） |
| 写真アップロード | 複数同時選択・フェーズ一括設定（500枚まで） |
| 台帳ビュー | 写真一覧とコメント入力を1画面に表示 |
| コメントテンプレート | システムテンプレート・カスタムテンプレート・最近使ったコメント |
| 施工フェーズタグ | 施工前・施工中・施工後の分類 |
| 写真並び替え | 台帳内での順序変更（▲▼ボタン） |
| 広告表示 | バナー広告（非侵害的な位置に配置） |

### 2.2 Proプラン機能（月額300円）

| 機能 | 説明 |
|---|---|
| 上記全機能の無制限利用 | プロジェクト・写真枚数の上限なし（50GB） |
| 広告非表示 | |
| **Excel出力** | 写真・コメント・番号を含む工事写真台帳をExcelで生成 |
| **削除時自動行詰め** | 写真削除時にコメント削除・行詰め・連番振り直しを自動処理 |
| PDF出力 | 工事写真帳形式のPDFを生成 |
| 写真自動圧縮 | アップロード時にWebP変換・ファイルサイズ削減 |
| 工事看板挿入 | 工事名・日付・会社名を写真に合成 |
| Google Drive連携 | プロジェクトごとに自動バックアップ・同期 |
| カスタムタグ | 工種・工程・撮影場所タグを追加作成 |

### 2.3 法人プラン（要見積 / 月20,000円〜）

| 機能 | 説明 |
|---|---|
| 複数ユーザー共有 | 閲覧者・編集者・管理者の権限管理 |
| 管理者ダッシュボード | 全ユーザーの使用状況一覧 |
| 独自テンプレート共有 | 会社統一フォーマットをメンバー全員が使用 |
| 請求書払い対応 | 月末締め翌月払い |
| 優先サポート | 翌営業日以内回答 |

### 2.4 Excel出力 詳細仕様

**出力内容（MVP版：固定2列テンプレート）**

```
表紙シート
  工事名・出力日・写真枚数

写真シート（2列固定レイアウト）
  ┌──────────────┬──────────────┐
  │   No.1       │   No.2       │
  │   [写真]     │   [写真]     │
  │   施工前     │   施工後     │
  │   2026-05-30 │   2026-05-31 │
  │   コメント   │   コメント   │
  └──────────────┴──────────────┘
```

**削除時の自動処理（Phase 2以降）**

```
写真を削除
  → 対応するコメントセルをクリア
  → 空になった行を上詰め（行シフト）
  → 空になったページを詰める
  → ページ番号を自動採番し直す
```

### 2.5 コメントテンプレート仕様

**3階層のテンプレート**

| 階層 | 内容 | 登録方法 |
|---|---|---|
| システムテンプレート | アプリが初期提供。施工フェーズ別に分類 | 変更不可 |
| カスタムテンプレート | ユーザーが登録。プロジェクト横断で使用 | 入力欄から1タップ保存 |
| 最近使ったコメント | 直近10件を自動記録 | 自動 |

**システムテンプレート初期値**

| 施工フェーズ | テンプレート一覧 |
|---|---|
| 施工前 | 全景（施工前）・補修範囲確認・既存状態確認・ひび割れ状況・劣化状況確認・墨出し確認 |
| 施工中 | 下地処理完了・清掃完了・プライマー塗布完了・材料充填完了・養生状態確認・中間検査 |
| 施工後 | 全景（施工後）・仕上がり確認・補修完了・清掃後確認・完成検査 |

## 6. 設計上の判断まとめ

| 判断 | 理由 |
|---|---|
| phase を photos に直接持つ | MVPでは施工フェーズのみ管理。タグテーブル経由だと結合が増えて複雑になる |
| sort_order は 1000 刻みの初期値 | フラクショナルインデックスで並び替えのたびに全件更新する処理を回避 |
| 写真番号は動的計算（DBに持たない） | 削除・並び替えで自動的に詰まる。連番保証のためのトリガー不要 |
| recent_comments を別テーブルで管理 | photos テーブルのコメント履歴とは切り離し、ユーザー横断で使い回せる |
| コメントテンプレートを AI より先に実装 | 実装コストが低い・現場の言葉で書ける・AI の学習データになる |
| Excel生成をクライアントサイドで実装 | サーバー不要。Supabase Edge Function のコストが発生しない |
| サムネイルをアップロード前にブラウザで生成 | サーバー処理不要。imageUtils.ts で Canvas を使って生成 |

## 8. レスポンシブ設計方針

最終更新: 2026-06-03

### 8.1 デバイス別利用シナリオ

このアプリは **同一URL・同一アカウント** でスマホとPCの両方から使える。
デバイスごとに「得意な操作」が異なるため、同じ画面でもレイアウトと操作体系を切り替える。

| デバイス | 主な利用場所 | 主な作業 |
|---|---|---|
| スマホ（〜767px） | 現場 | 写真撮影・アップロード・フェーズ一括設定 |
| PC（768px〜） | 帰社後・事務所 | 台帳ビュー確認・コメント入力・並び替え・Excel出力 |

### 8.2 ナビゲーション構造の違い

**スマホ：ボトムナビゲーション**

```
┌─────────────────────────────────┐

├─────────────────────────────────┤
```

**PC台帳ビューの強化点**

| 要素 | PC専用仕様 |
|---|---|
| テーブルレイアウト | 全カラムを1行で表示（No・写真・フェーズ・日時・コメント・操作） |
| コメント入力欄 | 横幅が広いためテキストが見やすい。Enterキーで次行へ移動 |
| テンプレートボタン | [T]は常時表示。ホバーでプレビュー表示 |
| ▲▼ボタン | 常時表示（スマホでは行が狭いため省スペース表示） |
| 削除ボタン | [×]を行右端に常時表示（スマホでは長押しメニュー経由） |
| Excel出力 | ヘッダー右上に大きく配置。1クリックで起動 |
| 写真追加 | ヘッダーにボタン配置 + ドラッグ&ドロップをテーブル外に直接ドロップ可 |
| キーボード操作 | Tab/Shift+Tabでコメント入力欄を順番に移動 |

### 8.5 ブレークポイント定義

| ブレークポイント | 幅 | 対象デバイス | 主な変化 |
|---|---|---|---|
| `sm` | 〜767px | スマートフォン | ボトムナビ表示・サイドバー非表示・1列カード・台帳簡略表示 |
| `md` | 768px〜1023px | タブレット | サイドバーアイコン表示・2列カード・台帳通常表示 |
| `lg` | 1024px〜 | PC・大画面 | サイドバーフル表示・3-4列カード・台帳テーブル全表示 |
